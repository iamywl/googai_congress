MetricLens AI

Forecast server load, compute the smallest resources that keep the SLO

A rightsizing-optimization platform that forecasts CPU·memory load with a lightweight time-series model and computes, via integer programming, the smallest allocation that preserves the SLO. It runs on CPUs alone (no GPU) and never sends data off-site, so it works self-contained in air-gapped / regulated on-premises environments.

🖥️ Live dashboard 🔌 API (Swagger) 📊 Presentation (slides)
The problem we solve

Over-provisioning

Per the Azure trace (SOSP'17), ~60% of VMs average under 20% CPU. Most allocated capacity sits idle, wasting cost and power.

A tooling gap

Public-cloud optimizers must send telemetry off-site, so they are unusable in air-gapped environments. On-prem monitors stop at reporting.

Our answer

With no external dependency and no GPU, quantify "how much can we cut and still keep the SLO" from the data.

How it works (4 stages)

① Forecast

Seasonal-trend decomposition + AR(1) residual correction estimate future load and a 95% prediction interval. A white-box model using the standard library only.

② Robust peak (p95)

Use the p95 as the peak instead of the max, preventing over-provisioning caused by lone spikes.

③ Integer-program optimization

Solve exactly, by exhaustive enumeration, for the smallest integer allocation under the headroom constraint (integer vCPU, 256 MB memory blocks).

④ Resize + audit

Map to a GCP machine type, resize the real instance, and persist every decision to an audit log.

peakp95 · safety_margin ≤ target_util · allocation

The core constraint — the smallest integer allocation that satisfies it is the minimum-cost configuration that keeps the SLO.
