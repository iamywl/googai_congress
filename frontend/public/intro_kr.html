서버 부하를 예측하고, SLO를 깨지 않는 최소 자원을 산출한다

경량 시계열 모델로 서버의 CPU·메모리 부하를 예측하고, 정수계획법으로 SLO(서비스 목표)를 보장하는 최소 자원 구성을 계산하는 리사이징 최적화 플랫폼이다. GPU 없이 CPU만으로 동작하며, 외부로 데이터를 보내지 않아 망분리·규제 온프레미스 환경에서도 자립적으로 쓸 수 있다.

🖥️ 라이브 대시보드 🔌 API (Swagger) 📊 발표자료(슬라이드)
해결하는 문제

과프로비저닝

Azure 트레이스(SOSP'17) 기준 VM의 약 60%가 평균 CPU 20% 미만. 할당 자원의 대부분이 유휴 상태로 비용·전력을 낭비한다.

도구의 공백

퍼블릭 클라우드 옵티마이저는 텔레메트리를 외부로 보내야 해 망분리 환경에서 사용 불가. 온프레 모니터는 사후 리포팅에 그친다.

우리의 답

외부 의존성·GPU 없이 데이터에 근거해 "얼마나 줄여도 SLO를 지키는지"를 정량 산출한다.

동작 원리 (4단계)

① 예측

계절-추세 분해 + AR(1) 잔차 보정으로 미래 부하와 95% 예측구간을 추정한다. 표준 라이브러리만 사용하는 화이트박스 모델.

② 강건 피크 (p95)

최댓값 대신 p95를 피크로 사용해 단발 스파이크로 인한 과다 할당을 막는다.

③ 정수계획 최적화

헤드룸 제약 하에서 최소 정수 할당을 전수 열거로 정확히 푼다(vCPU 정수·메모리 256MB 블록).

④ 리사이즈 + 감사

GCP 머신타입에 매핑해 실제 인스턴스를 리사이즈하고 모든 결정을 감사 로그로 영속 기록한다.

peakp95 · safety_margin ≤ target_util · allocation

핵심 제약식 — 이를 만족하는 가장 작은 정수 할당이 곧 SLO를 지키는 최소 비용 구성이다.

검증된 결과
6 / 6seasonal-naive 능가 (RMSE)
5 / 6MASE < 1 (스케일 불변 우수)
4 / 6DM 검정 p<0.05 유의
0.93–0.98예측구간 커버리지 PICP

MASE·Diebold–Mariano 검정·PICP/MPIW로 논문 수준 평가. 데이터는 Azure·Alibaba·Barroso & Hölzle 트레이스에 정렬된 합성 워크로드(6종 아키타입).

실제 GCP 연동

실측 모니터링

Cloud Monitoring으로 실제 인스턴스의 CPU·메모리를 적재.

실제 리사이즈

Compute Engine API로 머신 타입을 실제 변경(stop→change→start).

안전장치

월 ₩300k 예산 가드 + 보호 인스턴스 denylist.
